return vs console.log

return: the way to return data from a function.


console.log: feature for debugging purposes. Used to check if a value exists. Console.log will show the value of an object and then forget about it. If you want to hold on to the value and do something with it, then you need the return keyword.

Logical Operators AND and OR

&& and ||

  1. used to combine two logical expressions into one larger logical expression
  2. if both expressions evaluate to true, then the whole expression is true
  3. if both evaluate to false, then the whole expression is false
    
      var xPos = 300;
      var yPos = 150;

      function sendWarning(x, y) {
          if ((x < xPos) && (y < yPos)) {
              alert("Adjust the position");
          } else {
              alert("Things are fine!");
          }
      }

      sendWarning(500, 160);
      sendWarning(100, 100);
      sendWarning(201, 149);
There are three conditions being tested here. The first one is whether x is less than xPos. The second one is whether y is less than yPos. The third is seeing whether the first statement and the second statement both evaluate to true to allow the && operator to return a true as well. You can chain many series of conditional statements together depending on what you are doing. The tricky thing, besides learning what all the operators do, is to ensure that each condition and subcondition is properly insulated using parentheses.

Truthy and Falsey

Every value in JavaScript has an inherent boolean value. When that value is evaluated in the context of a boolean expression, the value will be transformed into that inherent boolean value.

All of the falsey values:

  1. the Boolean value false
  2. the null type
  3. the undefined type
  4. the number 0
  5. the empty string ("")
  6. the odd value NaN (stands for "not a number", check out the NaN MDN article)

That's right, there are only six falsy values in all of JavaScript!

All the truth values

    Essentially, if it's not in the list of falsy values, then it's truthy!

Short Circuiting

As logical expressions are evaluated left-to-right, they are tested for possible "short-circuit" evaluation using the following rules:

  1. false && (anything) is short-circuit evaluated to false

  2. true || (anything) is short-circuit evaluated to true
For example, in the conjunction x && y, if x evaluates to false, the whole expression will be false, regardless of what the other argument evaluates to. Similarly, for a disjunction x || y, if the first argument evaluates to true, the whole expression will be true, regardless of what the other argument evaluates to. They short circuit (stop) as soon as they have a result, but they return the value that stopped the processing (not necessarily tru or false)

Ternary operator

if/else statement

    
      var day = true;
      if(day){
        alert('It is daytime')
      } else{
        alert('It is nighttime')
      }
      // It is daytime

rewritten using the ternary operator

    
      var day = true; //conditon
      alert(day ? 'It is day-time' : 'It is night-time') // It is day-time
    
  
    
      var day = true; //conditon
      alert(day ? 'It is day-time' : 'It is night-time') // It is day-time
    
  
    
      - ? - means IF
      - : - means Else
