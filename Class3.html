Escaping Quotes and Apostrophes in Strings

Since quotation marks are used to denote strings, special considerations must be made when using apostrophes and quotes in strings.

Attempting to use an apostrophe in the middle of a single-quoted string, for example, will end the string, and JavaScript will attempt to parse the rest of the intended string as code. The same would apply to attempting to use quotes in a double-quoted string. 

              
                var brokenString = 'I'm a broken string';
                console.log(brokenString);

Options for Escape

  2. Escape characters

Escape Character ( \ )

Use \' or \"

  • prevents JavaScript from interpreting a quote as the end of the string
  • safely inserts quote without any fear of breaking the string

Data Type: Boolean

Comparing strings and strict equality...returns Boolean

Boolean:  This is always one of two words. Either true or false, with no quotations.

            
              “Yes” == “yes”...is FALSE, when you compare strings case matters.
              But, 10 == “10”...is TRUE because there is Implicit type coercion

Implicit Type Conversion (aka: Coercion)

Coercion: noun the practice of persuading someone to do something by using force or threats.


If you use a data type that JS did not expect, it tries to make sense of the operation rather than report an error.

              
                console.log('1' > 0)
                //true
                //the string '1' is converted to a number 1

Type coercion can lead to unexpected values in your code (and also cause errors).

              
                console.log( 12 + 34)

                console.log('12' + 34)

Best practice:

  1. Use === and !== to check if two values are equal
  2. strict equality(===) and strict inquality (!==) operators will check if the type and the value is the same
